1. y 2. Las Reglas del Juego

🤖 1. Google AI Studio

Como ya vimos, seleccionamos esta herramienta por comodidad y porque nos permite tener un entorno de chat avanzado donde es muy fácil pedir varias correcciones y ajustes al código generado.

🎨 2. Solo Frontend

Recuerda: la IA creará el aspecto visual (Frontend), no hace Backend. Las acciones como procesar pagos o guardar datos en bases de datos hay que hacerlas aparte o poniendo enlaces a herramientas externas (ej. Google Forms, Stripe).

Paso 3 Creamos la aplicación

Con Google AI Studio abierto, nos toca modificar el siguiente prompt para ajustarlo a tu negocio . Fijáte en las preguntas del apartado dos y respóndelas. Copia y pega:

📄 Estructura de la Landing Page
Actúa como un experto en diseño UX/UI y desarrollo frontend. Tu tarea es crear una landing page de alta conversión siguiendo estrictamente la estructura visual que te describo a continuación. 1. ESTRUCTURA DEL DISEÑO: Hero Section: Lado izquierdo: Título principal grande, subtítulo claro y un botón de llamada a la acción (CTA) llamativo. Lado derecho: Espacio reservado para una imagen o ilustración relevante. Debajo, una lista de 3 puntos destacados (beneficios rápidos). Sección de Beneficios: Grid de 3 columnas, cada una con un título, icono y una breve descripción. Debajo, un botón CTA centrado. Sección de Autoridad: Un bloque horizontal con fondo de color sólido para resaltar elementos de confianza (logotipos, métricas, frases de autoridad). Sección de Prueba Social: Dos testimonios con foto (circular), nombre y opinión del cliente. Sección Final (Last Call): Bloque de cierre con fondo oscuro, un título que refuerce la oferta y un botón CTA final. 2. INFORMACIÓN A PERSONALIZAR (Inputs del usuario): Por favor, usa esta información para completar el contenido de la landing: Propósito de la landing: Título principal: Subtítulo: Texto de los beneficios (3 puntos): CTA Principal (Texto del botón): Enlace de destino (Link de pago/formulario): Colores del diseño: 3. REQUISITOS TÉCNICOS Y DESPLIEGUE (Obligatorio): Para que este proyecto sea desplegable en GitHub Pages sin errores, debes configurar lo siguiente: Archivo vite.config.ts: Asegúrate de incluir la propiedad base: '/NOMBRE_DEL_REPO/'. (Pídeme el nombre del repo si es necesario). Archivo package.json: Verifica que el script build sea "vite build". Workflow de despliegue: Crea el archivo .github/workflows/deploy.yml con el siguiente contenido estándar actualizado (para evitar errores de versiones obsoletas): name: Deploy to GitHub Pages on: push: branches: - main permissions: contents: read pages: write id-token: write concurrency: group: "pages" cancel-in-progress: false jobs: build: runs-on: ubuntu-latest env: FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true steps: - name: Checkout uses: actions/checkout@v4 - name: Setup Node uses: actions/setup-node@v4 with: node-version: '20' cache: 'npm' - name: Install dependencies run: npm install - name: Build run: npm run build - name: Setup Pages uses: actions/configure-pages@v4 - name: Upload artifact uses: actions/upload-pages-artifact@v3 with: path: './dist' deploy: environment: name: github-pages url: ${{ steps.deployment.outputs.page_url }} runs-on: ubuntu-latest needs: build steps: - name: Deploy to GitHub Pages id: deployment uses: actions/deploy-pages@v4 Por favor, comienza analizando la estructura y dime cuando estés listo para recibir los datos de personalización.

Edición y Ajustes Ágiles

3.2 Activando el focus mode

Para editar cosas específicas de la landing, podemos seleccionar el icono de una flecha en un cuadrado que vemos en el chat. Esto activará el focus mode, un complemento que nos permitirá seleccionar cosas de la web para que el chat pueda entender que quieres un cambio exactamente ahí.

Transición a Producción

  • Paso 4 Creamos cuenta en GitHub

    Si aún no la tienes, ve a GitHub.com y regístrate. Será nuestro alojamiento gratuito. Entra aquí https://github.com/

  • Paso 5 Descargamos el repositorio

    Desde Google AI Studio, descargamos todos los archivos del proyecto a nuestro ordenador (suele venir en un archivo ZIP). Descomprímelo en una carpeta local.

Paso 6 Lo subimos a GitHub

Sigue estos pasos EXACTOS antes de hacer el commit inicial:

  1. Crea un repositorio nuevo en GitHub y ábrelo con GitHub Codespaces (botón verde "Code" > pestaña Codespaces) para tener tu editor online listo.
  2. Subimos sólo los archivos de la carpeta general (no las subcarpetas directamente sueltas, respeta la estructura).
  3. Borramos el archivo env example.
  4. Borramos el archivo readme.
  5. Cambiamos el titulo de la web en index.html quitando lo de "My Google AI Studio App" y poniendo un título chulo para nuestra web.
  6. Vamos a vite.config.ts y buscamos donde pone "base:/NOMBRE_DEL_REPO/". Nos aseguramos de cambiar lo que hay entre las barras / / para poner el nombre de nuestro repositorio de GitHub.
  7. Subimos las carpetas.
  8. Vamos al control de código fuente y hacemos un push.

El Toque Final

Paso 7 Activamos Pages

En tu repositorio de GitHub, ve a Settings > Pages. En la sección "Build and deployment", selecciona GitHub Actions como source.

Paso 8 Rerun Actions

Ve a la pestaña Actions en el menú superior de GitHub. Selecciona el workflow fallido o en pausa y dale a Re-run jobs. ¡Espera unos minutos y tu web estará online!
